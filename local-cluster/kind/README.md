# kind (Kubernetes IN Docker)

Runs a Kubernetes cluster using Docker containers as the nodes. Fast to create and destroy, and supports multi-node clusters, which makes it a favourite for CI and for testing things like pod scheduling and node failures.

**When to use:** CI pipelines, multi-node testing, quick throwaway clusters.

**Prerequisites:** Docker.
